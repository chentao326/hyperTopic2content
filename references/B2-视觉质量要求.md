# B2 MP4 视觉质量要求

当用户说「重新出视频」或「做的不好」时，通常原因：之前的 MP4 只有纯色背景+字幕，缺少视觉内容。

## 修正方案

用 HyperFrames 创建章节式视觉幻灯片（类似 web-video-presentation 风格），包含：

- 每个核心论点对应一个独立视觉场景（scene）
- 关键数据以大号字展示（如 1.35万亿、61%份额）
- 对比关系用 VS 卡片并排展示
- chapter-tag 标注章节序号（如 01 · 大逆转）
- 底部字幕同步覆盖（从 transcript.json 读取）
- 统一的设计主题（配色/字体/装饰线）

## 场景结构建议

| 场景类型 | 内容 | 示例 |
|---------|------|------|
| 标题页 | 视频标题 + 副标题 | "SK海力士逆袭之路" |
| 数字冲击页 | 核心数字大号展示 | 1.35万亿、26年 |
| 时间线页 | 关键事件推进 | 1983→2004→2012→2022 |
| 对比页 | VS 并排卡片 | SK海力士 vs 三星 |
| 引用页 | 金句突出展示 | "赌的不是海力士..." |
| 数据页 | 市场份额对比 | 61% vs 17% |
| 教训页 | 核心观点/结论 | "大公司的诅咒" |

## 原则

- 宁可减少章节数量（6-8 个 scene），也不要只有字幕没有画面
- 每个 scene 停留 5-10 秒（配合音频节奏）
- 字幕始终覆盖在底部，与场景动画独立运行
- 字幕使用半透明背景，不影响下方视觉内容
