# 提炼口播稿的 LLM Prompt 模板

Phase 4 Step A 使用。从 Phase 3 公众号文章提炼口播稿，同时产出 script_full.txt（B2 TTS 用）、script.md（B1 用）和 outline.md（B1 规划用）。

## Prompt

```
你是 B 站口播视频脚本作者。基于以下公众号文章，提炼一段口播稿。

要求：
- 短视频版总字数 120-220 字（语速 3-4 字/秒，约 30-60 秒音频）
- 完整版总字数 300-600 字（约 90-180 秒音频；更长内容拆成多段）
- 开头 1-2 句钩子（提问/反差/震撼数字）
- 中间 2-3 个核心论点（每个必须有数据或具体事实支撑）
- 结尾 1-2 句收尾（留思考或展望，不单纯总结）
- B 站口播风格：短句、不堆术语、口语化
- 禁用：冒号、双引号、破折号、学术黑话
- 保留原文 1-2 个金句的意思，但改写成口语化表达，不使用引号
- 禁止出现：说白了/本质上/换句话说/值得注意的是/从这个角度

输出三份：
1. 完整口播稿（纯文本，供 TTS 用）
2. 按章节拆分的版本（每章 2-3 个自然段，供网页演示用）
3. 网页演示开发规划（信息池 + 素材清单 + 每步估时）

==== 公众号文章 ====
[Phase 3 公众号文章全文]
==== 文章结束 ====

输出格式：
---FULL---
[完整口播稿纯文本]
---CHAPTERS---
[按章节拆分]
---OUTLINE---
[信息池]
- 关键事实：
- 关键数据：
- 关键判断：

[素材清单]
- 视觉元素：
- 数据图表：
- 动效建议：

[Step 估时]
- Step 1：
- Step 2：
- Step 3：
```

## 长度控制规则

| Phase 2 报告字数 | 口播稿推荐字数 | 估计音频时长 | B2 输出推荐 |
|----------------|------------|------------|-----------|
| <4000 字 | 120-220 字 | 30-60s | 短视频版 |
| 4000-8000 字 | 220-400 字 | 60-120s | 短视频版或完整版 |
| >8000 字 | 400-600 字 | 100-180s | 完整版 |

口播稿过长时（>600 字），拆为 2-3 段短视频脚本，每段 120-220 字。

## 自动修正规则

生成后自动检查：
- [ ] 冒号「：」全部改为逗号
- [ ] 无破折号「——」
- [ ] 无双引号（如果在 prompt 中指定禁用）
- [ ] 开头有钩子（不是直接说"今天我们来聊聊"）
- [ ] 结尾有收尾（不是"以上就是本次内容"）
- [ ] 每句话 ≤ 20 字（长句要拆）
- [ ] 全文无英文术语（或用中文注释替换）
